Machine Learning & Deep Learning

What is Machine Learning?

Machine Learning (ML) is a way to make computers learn from data — without explicitly telling them every rule. Instead of programming "if X then do Y," you give the computer lots of examples and let it figure out the patterns on its own.

🧠 Analogy: Teaching a child to recognize cats. You don't explain "a cat has fur, whiskers, four legs…" Instead, you show them 1000 pictures of cats and 1000 pictures of dogs. Eventually they recognize cats on their own — that's ML!

3 Main Types of ML

📌 Supervised Learning

Trains on labeled data (input + correct answer). Learns to predict on new data.
Example: Spam email detection

🔍 Unsupervised Learning

Trains on unlabeled data. Finds hidden patterns or groups.
Example: Grouping customers by behavior

🎮 Reinforcement Learning

Learns by trial and error — gets rewards for good actions.
Example: Training a game-playing AI

🔀 Semi-Supervised

Mix of labeled + unlabeled data. Useful when labeling data is expensive.
Example: Medical image classification

What is Deep Learning?

Deep Learning (DL) is a sub-field of ML that uses multi-layered artificial neural networks (inspired by the human brain) to automatically learn features from raw data. The "deep" in deep learning refers to the many hidden layers in the network.

🧠 Analogy: In regular ML, a human engineer has to manually pick out features from a photo (edges, colors, shapes). In Deep Learning, the network figures out those features by itself — layer by layer, like peeling an onion.

Gradient Descent

What Problem Does It Solve?

When training a neural network, it makes predictions and compares them to the correct answer using a Loss Function (a number that measures how wrong the model is). Gradient Descent is the algorithm that minimizes this loss — it's how the model learns and improves.

⛰️ Classic Analogy: Imagine you're blindfolded on a hilly mountain and you want to reach the lowest valley. You can't see far, but you can feel which direction is downhill under your feet. So you take a small step in the steepest downhill direction — then repeat. Eventually you reach the bottom (minimum loss). That's Gradient Descent!

How Does It Work? (Step by Step)

  1. Start with random weights in the neural network.
  2. Feed training data through the network to get a prediction.
  3. Calculate the loss (how wrong the prediction is).
  4. Compute the gradient — which direction increases the loss most.
  5. Move in the opposite direction (downhill) by a small step.
  6. Update the weights and repeat until loss is minimized.

The Learning Rate

The "small step" size is called the Learning Rate (α). It controls how fast or slow the model learns.

Too High Learning Rate
⚠️ Overshoots the minimum — bounces around, never converges
Too Low Learning Rate
🐢 Very slow — takes forever to reach the minimum

The Gradient Descent Formula

new_weight = old_weight − α × gradient_of_loss

w = w − α · (∂L / ∂w)

Where α is the learning rate and ∂L/∂w is the derivative (slope) of the loss with respect to each weight.

Types of Gradient Descent

Batch GD

Uses the entire dataset to compute one update. Accurate but slow for large data.

Stochastic GD (SGD)

Uses one sample at a time. Very fast but noisy updates.

Mini-Batch GD

Uses a small batch (e.g., 32 samples). Best of both worlds — most widely used.

Adam Optimizer

An advanced form of GD that adapts the learning rate automatically. Very popular in practice.

Key Insight: Gradient Descent is the backbone of training all neural networks. Without it, neural networks cannot learn from data. It uses the backpropagation algorithm to efficiently compute gradients across all layers.

Autoencoders

What Is an Autoencoder?

An Autoencoder is a type of neural network that learns to compress data into a smaller form, and then reconstruct it back to the original. It does this completely unsupervised — no labels needed!

📦 Analogy: Imagine you have a massive HD photo you want to send to a friend. You compress it into a tiny zip file (encoding), send it, and your friend unzips it (decoding) to get the photo back. The autoencoder learns the best way to zip and unzip data automatically.

Three Main Parts

🔽 Encoder

Compresses the input into a smaller "summary" (latent space). Hidden layers shrink in size.

⚪ Bottleneck

The middle layer — the compressed representation of the data. This is the "essence" of the input.

🔼 Decoder

Takes the compressed form and tries to rebuild the original input. Hidden layers grow back in size.

How is it Trained?

The autoencoder is trained by comparing its output (reconstructed data) to the original input. The goal is to minimize the reconstruction loss — the difference between what came in and what came out.

  • MSE (Mean Squared Error) — for continuous data like images
  • Binary Cross-Entropy — for binary (0/1) data

What Are Autoencoders Used For?

  • Dimensionality Reduction — like PCA but more powerful
  • Noise Removal — clean up noisy images or data
  • Anomaly Detection — find unusual patterns (fraud, defects)
  • Data Compression — store data more efficiently
  • Generative AI — generate new data similar to training data (VAE)

Types of Autoencoders

1. Denoising Autoencoder

This type is trained with intentionally corrupted (noisy) input. The job of the network is to remove the noise and reconstruct the clean original.

🎨 Analogy: Give someone a blurry, scratched photo and ask them to restore it to the original. Over time, they get very good at fixing photos — that's a denoising autoencoder!
✅ Advantage
Learns robust, meaningful features. Can be used for data augmentation.
⚠️ Disadvantage
Choosing the right noise level is tricky. Can lose useful original information.

2. Sparse Autoencoder

This autoencoder has more neurons than needed, but forces most of them to be inactive (zero) at any given time. This is called "sparsity" — only a few neurons fire for each input.

💡 Analogy: Imagine a room full of light bulbs, but only 5% are allowed to be on at any time. The network learns to pick the most meaningful 5% of features for each input.
✅ Advantage
Filters noise, learns meaningful sparse features.
⚠️ Disadvantage
Sensitive to hyperparameters, higher computational cost.

3. Variational Autoencoder (VAE)

A VAE is a generative model. Instead of compressing data into fixed values, it learns a probability distribution (a range of possible values) in the latent space. This lets it generate new data that resembles the training data.

🎭 Analogy: A regular autoencoder learns "this face = these exact numbers." A VAE learns "this face = somewhere in this region of face-space." You can then sample random points from that region to generate new faces you've never seen before!
✅ Advantage
Can generate new synthetic data. Great for anomaly detection and exploration.
⚠️ Disadvantage
Uses approximations — introduces some error. Generated samples may lack diversity.

4. Convolutional Autoencoder

This autoencoder uses Convolutional Neural Network (CNN) layers instead of regular dense layers. It's specifically designed for image data, since CNNs are great at understanding spatial patterns.

  • Encoder uses Conv layers to progressively compress the image
  • Decoder uses transposed Conv (deconv) layers to reconstruct the image
  • Excellent for image denoising, compression, and inpainting
Summary Table:

Denoising AE → Remove noise from data
Sparse AE → Learn key features with minimal activations
Variational AE → Generate new data (probabilistic)
Convolutional AE → Best for image data

Convolutional Neural Networks (CNN)

What is a CNN?

A Convolutional Neural Network (CNN) is a type of deep learning model specifically designed for images. Instead of looking at the whole image at once, it scans small regions using filters (kernels) to detect features like edges, shapes, textures, and eventually full objects.

🔎 Analogy: Imagine reading a large painting with a magnifying glass — you scan small areas one at a time, noticing individual details (brushstrokes, colors, shapes) before putting it all together to understand the whole image. That scanning process is exactly what a CNN's convolutional layer does!

The 5 Layers of a CNN (Deep Convolutional Model)

  1. Convolutional Layer + ReLU — Applies filters to detect features (edges, curves). ReLU removes negative values, adding non-linearity.
  2. Pooling Layer — Shrinks the feature maps (usually max pooling) to reduce computation and retain dominant features.
  3. Fully Connected Layer — Flattens and connects all features for final classification — acts like a regular neural network.
  4. Dropout Layer — Randomly turns off some neurons during training to prevent overfitting.
  5. Activation Function Layer — Final layer that outputs the class probabilities (usually Softmax for multi-class).

Why Are CNNs So Powerful for Images?

  • Local receptive fields — filters detect local patterns (edges) regardless of where they appear in the image
  • Parameter sharing — same filter is reused across the image → far fewer parameters than a regular network
  • Translation invariance — can recognize a cat whether it's top-left or bottom-right in the image
  • Hierarchical features — early layers detect edges, later layers detect faces, objects

Famous CNN Architectures

ModelYearKey Feature

Real-World CNN Applications

  • Image classification (Google Photos, Instagram filters)
  • Object detection (self-driving cars, security cameras)
  • Medical imaging (detecting tumors in X-rays)
  • Visual search (Google Lens, Amazon product search)
  • Recommendation engines based on image similarity

CNN Datasets (CIFAR, MNIST, ImageNet…)

Why Do CNN Datasets Matter?

CNNs are only as good as the data they're trained on. Benchmark datasets are standardized collections of labeled images used by researchers worldwide to train and compare CNN models fairly.

📚 Analogy: Think of these datasets as the "textbooks" for CNNs. All students (models) study from the same book, so you can fairly compare who learned better.

Most Important CNN Datasets

MNIST

70,000 handwritten digit images (0–9). The "Hello World" of deep learning. Black & white, 28×28 pixels. Perfect for beginners.

CIFAR-10 / CIFAR-100

60,000 color images (32×32). CIFAR-10 has 10 classes (cat, dog, car…). CIFAR-100 has 100 classes. Harder than MNIST.

ImageNet

14 million+ high-res images across 20,000+ categories. The gold standard for CNN benchmarks. Home of the ImageNet Challenge (ILSVRC).

MS COCO

200,000+ images with object detection, segmentation, and captions. Used for complex tasks beyond simple classification.

How Are Datasets Used in CNN Training?

  1. Training Set (~70%) — The model learns from these images and their labels.
  2. Validation Set (~15%) — Used during training to tune hyperparameters and monitor performance.
  3. Test Set (~15%) — Final, unseen evaluation to measure the model's real-world performance.

Data Preprocessing Steps

  • Normalization — Scale pixel values from 0–255 to 0–1 for faster training
  • Augmentation — Flip, rotate, crop images to artificially expand the dataset
  • Resizing — Standardize all images to the same resolution (e.g., 224×224)
  • One-hot encoding — Convert labels to vectors (e.g., "cat" → [0,1,0,0,0])
CMM Note: "CMM Dataset" in your course likely refers to the benchmark datasets used specifically in CNN labs and experiments — often CIFAR-10 or MNIST — which are the most commonly used in academic demonstrations of CNN performance.

Deep Neural Network — Structure & Weights

Structure of a Deep Neural Network

A Deep Neural Network (DNN) is made of multiple layers of neurons stacked on top of each other. Each layer transforms data a little more, allowing the network to learn increasingly complex patterns.

⬛ Input Layer

Receives raw data (pixel values, numbers, etc.). One neuron per feature. Does no computation.

🔲 Hidden Layers

One or more layers between input and output. Each layer extracts more abstract features. The "deep" part!

✅ Output Layer

Produces the final prediction. Number of neurons = number of classes (for classification).

What Are Weights?

Weights are the learnable parameters of a neural network — the numbers that get adjusted during training to make the model better. Each connection between two neurons has one weight.

⚖️ Analogy: Imagine you're a judge scoring a competition with 5 criteria. You assign different importance scores (weights) to each criterion. A confident judge gives high weight to "skill" and low weight to "costume." The network does the same — it learns which input features matter most for predicting the output.

The Computation Inside a Neuron

Each neuron performs two steps:

Step 1 — Weighted sum: z = w₁·x₁ + w₂·x₂ + ... + wₙ·xₙ + b

Step 2 — Activation function: output = f(z) ← e.g., ReLU, Sigmoid, Tanh

Where w = weights, x = inputs, b = bias, f = activation function.

What is a Bias?

The bias (b) is an extra parameter added to each neuron. It lets the network shift the activation function left or right — giving it more flexibility to fit data that doesn't pass through the origin.

📏 Analogy: In the equation of a line y = mx + c, the constant c shifts the line up or down. Bias is exactly that — it shifts the decision boundary of the neuron.

Common Activation Functions

NameFormulaUsed Where?

Sigmoid1 / (1 + e⁻ᶻ)Binary output (0 or 1)

How Weights Are Learned (Training)

  1. Initialize weights randomly (small random values).
  2. Forward Pass — feed input through the network, layer by layer, to get a prediction.
  3. Compute Loss — measure how wrong the prediction is.
  4. Backward Pass (Backpropagation) — use the chain rule to compute how much each weight contributed to the error.
  5. Update Weights using Gradient Descent: w = w − α · (∂Loss/∂w).
  6. Repeat steps 2–5 for many epochs until loss is minimized.
Key Insight: In a DNN with millions of weights (like ResNet-152), backpropagation + gradient descent adjusts every single weight simultaneously in a mathematically precise way. This is why training large models requires powerful GPUs and days or weeks of computation.

Overfitting vs Underfitting

Underfitting
Model is too simple — doesn't learn enough from training data. High error on both train and test sets.
Overfitting
Model memorizes training data but fails on new data. Low train error, high test error. Fix: Dropout, Regularization.

Linear Classifiers — Numericals

Quick Recap: How a Linear Classifier Works

A linear classifier makes predictions using a weighted sum of input features plus a bias:

z = w₁·x₁ + w₂·x₂ + ... + wₙ·xₙ + b

If z ≥ 0 → Class +1 (Positive class) If z < 0 → Class −1 (Negative class)

The decision boundary is the line/plane where z = 0 exactly — neither class, right on the border.

📝 Numerical 1 — Classify a Point

Question
A linear classifier is defined as:

z = 2x₁ − x₂ − 3

Classify the data point  (x₁, x₂) = (2, 4)

Solution — Step by Step

  1. Write down the classifier formula:
    z = 2x₁ − 1·x₂ − 3
  2. Substitute the values x₁ = 2, x₂ = 4:
    z = 2(2) − 1(4) − 3
  3. Perform the arithmetic:
    z = 4 − 4 − 3 = −3
  4. Apply the decision rule:
    z = −3 < 0 → Class = −1 (Negative class)
✅ Answer: The point (2, 4) is classified as Class −1, because plugging in the values gives z = −3, which is less than 0.

📝 Numerical 2 — Find the Decision Boundary

Question
For the classifier:

z = x₁ + 2x₂ − 4

Find the decision boundary equation.

Solution — Step by Step

  1. Understand what "decision boundary" means:
    The decision boundary is the line where the classifier is exactly on the fence — z = 0. Points on one side are Class +1, points on the other are Class −1.
  2. Set z = 0 in the classifier equation:
    x₁ + 2x₂ − 4 = 0
  3. Rearrange into a clean line equation:
    x₁ + 2x₂ = 4
✅ Answer: The decision boundary is the straight line  x₁ + 2x₂ = 4. Any point on this line gives z = 0 (on the boundary). Points where x₁ + 2x₂ > 4 → Class +1; points where x₁ + 2x₂ < 4 → Class −1.

📝 Numerical 3 — Verify Multiple Points

Question
Using  z = x₁ + 2x₂ − 4, classify these three points:

A = (0, 0)    B = (2, 1)    C = (4, 2)

Solution

PointCalculationz valueClass
A = (0, 0) z = 0 + 2(0) − 4 −4 Class −1
B = (2, 1) z = 2 + 2(1) − 4 0 On Boundary
C = (4, 2) z = 4 + 2(2) − 4 +4 Class +1
Observation: Point B lands exactly on the decision boundary (z = 0). In practice, the classifier may assign it to either class depending on the convention used (z ≥ 0 → +1 means B goes to Class +1).

🧠 Key Formulas to Remember

General linear classifier: z = w₁x₁ + w₂x₂ + b

Decision rule: z ≥ 0 → Class +1 z < 0 → Class −1

Decision boundary: Set z = 0 → w₁x₁ + w₂x₂ + b = 0

Exam Tip — How to Approach Any Linear Classifier Numerical

  1. Write out the classifier formula clearly.
  2. If classifying a point → substitute x₁, x₂ values and compute z.
  3. If finding boundary → set z = 0 and rearrange the equation.
  4. Apply the decision rule: z ≥ 0 is Class +1, z < 0 is Class −1.
